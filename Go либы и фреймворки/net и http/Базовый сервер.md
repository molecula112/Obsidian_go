```go
package main  
  
import (  
"fmt"  
"net/http"  
)  
  
type MyHandler struct{}  
  
func (h MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {  
	fmt.Fprintln(w, "hello world")  
}  
  
func main() {  
	handler := MyHandler{}  
	http.ListenAndServe(":8080", handler)  
}
```

Что здесь происходит?

Самая важная строка, эта строчка и создает сервер:
```go
http.ListenAndServe(":8080", handler)
```
Внутри происходит примерно это:
```
1. открыть TCP порт 8080  
2. ждать входящие соединения  
3. когда приходит соединение → создать goroutine  
4. прочитать HTTP запрос  
5. вызвать handler  
6. отправить ответ
```

Функция ожидает **интерфейс**:
```go
type Handler interface {  
	ServeHTTP(ResponseWriter, *Request)  
}
```

То есть сервер говорит:
```
дай мне что-то, что умеет обрабатывать HTTP запрос
``` 

Далее мы создали:
```go
type MyHandler struct{}
```

Это просто пустая структура, но теперь **MyHandler автоматически реализует интерфейс Handler**: 

```go
func (h MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request)
```

В Go **не нужно писать implements**.
Если есть нужный метод → интерфейс реализован.
