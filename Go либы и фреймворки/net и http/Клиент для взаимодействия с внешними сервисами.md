### 1. http.Method(быстро попробовать)
Старый вариант это напрямую вызывать методы http
```go
import (  
"fmt"  
"io"  
"net/http"  
)  
  
func main() {  
  
resp, err := http.Get("http://example.com/")  
if err != nil {  
	panic(err)  
}  
  
body, err := io.ReadAll(resp.Body)  
if err != nil {  
	panic(err)  
}  

defer resp.Body.Close()  --> Обрати внимание на это место, это очень важно
  
fmt.Println(string(body))  
```
Тут прикол в том что мы читаем по байтово ответ из TCP-соединения, и нам важно его закрыть чтобы не держать соединения и вернуть в connection pool:
```go
defer resp.Body.Close()
```
Проблемы этого подхода следующие:
- почти нет настройки
- трудно нормально управлять timeout, headers, context, retry

### 2. http.Client(production база)
Это просто удобная обертка над базовым http.Method
```go
client := &http.Client{
    Timeout: 5 * time.Second,
}

req, err := http.NewRequest(
    http.MethodGet,
    "https://api.example.com/users",
    nil,
)

if err != nil {
    return err
}

resp, err := client.Do(req)
if err != nil {
    return err
}

body, err := io.ReadAll(resp.Body)
if err != nil {
    return err
}

defer resp.Body.Close() --> в конце обязательно закрыть

```

Тут из прикольного:
Описываем запрос, третий параметр это тело запроса
```go
req, err := http.NewRequest(
    http.MethodGet,
    "https://api.example.com/users",
    nil,
)
```

Вот эта строчка самая главная, они и говорит, отправь запрос и получи ответ:
```go
resp, err := client.Do(req)
```


### 3. Свои обертки над http.Client
```go
type UserAPI struct {  
	client *http.Client  
	baseURL string  
}  
  
func (c *UserAPI) GetUsers() ([]User, error) {  
	req, _ := http.NewRequest(  
	http.MethodGet,  
	c.baseURL+"/users",  
	nil,  
)  
  
resp, err := c.client.Do(req)  
	if err != nil {  
	return nil, err  
}   
  
var users []User  
json.NewDecoder(resp.Body).Decode(&users)  

defer resp.Body.Close() 
  
return users, nil  
```
