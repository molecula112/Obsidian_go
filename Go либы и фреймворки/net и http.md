Библиотека для работы в вебе

Ключевая архитектура:
```
HTTP Server  
↓  
Handler interface  
↓  
ServeHTTP()  
↓  
твой код
```

Большинство GO фреймворков:
```
Gin  
Echo  
Fiber  
Chi
```
Это обертка над базовым net/http

Из приколов:
1) net/http не роутит по HTTP методу, но предоставляет константы для методов, то есть когда ты пишешь `mux.HandleFunc("/users", handler)` ServeMux проверяет лишь путь 
   `users`, он не смотри на GET, POST, PUT, DELETE, поэтому если хочешь сделать проверку и методов то нужно костылить.
   Связано: [[Костыль чтобы определить HTTP метод]]
2) net/http не умеет работать с path parameters, то есть он не понимает /users/123 и тд, то есть если ты зарегал вот так роут /users/ то значит все что после второго слеша тоже считается этим хэндлером.
   Связано: [[Костыль чтобы определить path parameters]]
