Данные вариант является устаревшим, после появления go modules

`Пакет - это группа связанных .go файлов в одной папке, которые компилируются вместе и образуют одну логическую часть программы.`

Или еще проще:
`package = папка с go-файлами, которые работают вместе`

---
### О пакете main
Пакет `main` это главный пакет всего приложения, он и делает программу исполняемой, то есть если программа должна запускаться, то она обязательно должна иметь функцию main. 
Пакет `main` нельзя импортировать, он предназначен только для запуска программы.
### Модификаторы доступов в пакетах:
`Если константа или функция с большой буквы - то данные вещи можно использовать из разных пакетов, если с маленькой то в рамках этого пакета.`

---
### Типы пакетов
1) std - это пакеты которые входят в стандартную библиотеку(fmt, context), располагаются они в GOPATH, в папке src. 
2) external - это пакеты которые у нас не скачаны, но мы подтягиваем их через `go get`, такие пакеты также попадают в GOPATH/src
3) local - это пакеты которые мы создаем сами

Немного о local, до появления go modules раньше было требование обязательно располагать свои пакеты в папке GOPATH/src, но такой подход не удобен

---
### Builtin функции(print/println)
У нас есть пакет builtin, нам не нужно импортировать его, туда входят типы, стандартные методы append/copy и тд

---

### Приколы при импортах
1) alias - можно задать кастомные названия для пакетов, их можно задать абсолютно любым пакетам и выглядит это так:
```go
import X "fmt"

func main() {
	X.Println("hello")
}   
```
2) импорт пакета для сайд эффекта - у некоторых пакетов есть функции, например, init(), и вот если мы заимпортим пакет `_ "fmt"` то мы по факту говорим что мы имортируем пакет для сайд эффекта и не собираемся вызывать ничего кроме init()
3) также можно заимпортить пакет через `.`, когда мы импортим так проект это значит что мы не хотим использовать его имя:
```go
import . "fmt"

func main() {
	Println("hello")
}
```
