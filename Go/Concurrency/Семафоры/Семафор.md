Семафор — это счётчик доступа к ресурсу

Ментальная модель:
```
есть N разрешений  
goroutine может взять разрешение  
если разрешений нет → нужно ждать  
когда разрешение освобождается → одна goroutine просыпается
```
---
#### Простая модель
Представь парковку на 3 места, машины (goroutines):
1. машина заезжает - место уменьшается
2. если мест нет - машина ждёт
3. когда машина уезжает  место освобождается
---
#### Виды семафоров
`Binary semaphore - значение может быть только 0 или 1`
То есть:
```
0 --> ресурс занят  
1 --> ресурс свободен
```
По сути это то же самое, что mutex
```
Acquire --> если 1, становится 0  
Acquire --> если 0, ждём  
Release --> становится 1
```
Прямая реализация - `var mu sync.Mutex`


`Counting semaphore - значение может быть больше 1`
Он хранит количество доступных ресурсов
```
semaphore = 3
```
Три goroutine могут работать одновременно:
```
Acquire → semaphore--  
Release → semaphore++
Если значение стало 0: следующие goroutine ждут
```
Прямая реализация - часто реализуют через `buffered channel`

---
#### Две операции семафора
Исторически они называются:
```
P()  --> взять ресурс (wait / acquire)  
V()  --> освободить ресурс (signal / release)
```
В современных API чаще:
```
Acquire()  
Release()
```

#### Чем семафор отличается от Mutex
Очень важная разница:
`Mutex`
1 разрешение
```
Lock  
Unlock
```

`Semaphore`
N разрешений
```
Acquire  
Release
```
---
#### Супер полезное правило
```
Mutex → защита данных  
Semaphore → ограничение параллелизма  
WaitGroup → ожидание завершения задач
```
