`WaitGroup` это инструмент для синхронизации, позволяющий дождаться завершения группы задач

Сам WaitGroup имеет всего 3 главных метода:
```go
var wg sync.WaitGroup

wg.Add()
wg.Wait()
wg.Done()
```

`wg.Add()` - увеличивает счетчик задач, то есть говорит `WaitGroup` сколько задач нужно дождаться
`wg.Wait()` - заставляет главную горутину дождаться завершения всех что добавлены в группу
`wg.Done()` - отпускает задачу из группы, считая что ее задача завершена, по факту уменьшает счетчик, аналог `wg.Add(-1)`

Зачем это нужно, есть задачка:
```go
func Task() {  
    for i := 0; i < 10; i++ {  
       go fmt.Println(i + 1)  
    }  
    fmt.Println("exit")  
}
```
Тут мы 10 раз в отдельной горутине просим вывести и посчитать небольшой примерчик, в данном случае результат будет непредсказуемый и главная горутина может завершиться раньше других
```
exit 

1
2
exit

4
1
3
exit

и тд
```

Решается как раз с помощью `WaitGroup`
```go
func Task() {  
	var wg sync.WaitGroup
	wg.Add(10) --> сколько горутин в функции используется, столько и добавляем
    
    for i := 0; i < 10; i++ {  
	   // или wg.Add(1) --> тут тоже можно добавлять
       go func(i int) {
	       defer Done() --> хорошая практика написать в самом начале и c defer
	       fmt.Println(i + 1)
       }(i)  
    } 
    
    wg.Wait() --> разблокируется только тогда когда задач у wg не останется
    fmt.Println("exit")  
}
```

В данном случае все горутины будут выполнены, но порядок выполнения будет все так же случайный, то есть данная фича нужна именно для того чтобы подождать пока все запущенные горутины будут выполнены
