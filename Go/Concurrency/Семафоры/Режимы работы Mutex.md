Mutex в Go работает в двух режимах

---
#### Normal mode
Это режим по умолчанию, когда `mutex` освобождается:
```
любая горутина может захватить lock
```
Даже новая горутина может перехватить `lock` раньше ожидающих

Это называется:
`lock stealing`

Плюс:
1. Больше throughput (`throughput = количество операций / время`), чем больше throughput - тем больше полезной работы выполняется
2. Меньше переключений scheduler
3. Меньше context switch

Минусы:
1. Возможен starvation
---
#### Starvation mode(fair mode)
Если горутина ждёт слишком долго (~1ms), `mutex` переключается в `fair режим`.

Теперь:
```
mutex передаётся строго следующей ожидающей горутине
```
Плюсы:
1. Справедливость (fairness)  - каждая горутина гарантированно получит mutex
2. Нет starvation

Минусы:
1. Падает `throughput` - `mutex` передаётся строго по очереди
