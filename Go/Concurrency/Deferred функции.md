Данные функции не вызываются в момент их вызова, а складываются в специальный defer stack функции и начинают вызываться после **return'а** основной функции в **обратном порядке**

```go
func main() {  
    defer fmt.Println(1)  
    defer fmt.Println(2)  
    fmt.Println("exit")  
}
```
Вывод будет
```
exit
2
1
```

Еще примерчик
```go
func summ(x, y int) (sum int) {  
    defer func() {  
       sum *= 2  
    }()  
    sum = x + y  
    return  
}

summ(2, 3)
```
Вывод будет
```
10
```

Важно понимать что defer не сработает вот так:
```go
defer func() {  
       sum *= 2  
    } --> без скобки
```
Потому что defer откладывает именно вызов!

Интересный кейс
```go
for i := 0; i < 10; i++ {
	defer func() {
		fmt.Println("first", i)
	}()
}
```
Данный код будет работать неправильно и будет выводить только 10

Cпособы исправления:
1) Тут параметры будут вычисляться сразу, в момент добавления функции в defer stack
```go
for i := 0; i < 10; i++ {
	defer fmt.Println("first", i)
}
```
2) Если мы сохраним в локальную переменную то функция будет помещаться в defer stack с нормальным стейтом локальной переменной 
```go
for i := 0; i < 10; i++ {
	k := i
	defer func() {
		fmt.Println("first", k)
	}()
}
```
3) Передать параметр в defer функцию, грубо говоря равносильно варианту 1
```go
for i := 0; i < 10; i++ {
	defer func(k int) {
		fmt.Println("first", k)
	}(i)
}
```
